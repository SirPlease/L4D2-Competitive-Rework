infected_control anti-baiter 时序理解图

这里按“anti-baiter 嵌在正常下一波倒计时里”的设计来画：它不是独立刷特系统，也不会把开波时间提前到基础刷特间隔之前； 它观察上一波处理完后，生还在等待下一波期间有没有正常推进，到点后再决定放行还是继续等机会。

当前波开始 StartWave 后刷出本波特感，同时记录本波起点和 anti-baiter baseline。
→
本波交火 生还处理特感；插件仍维护传送、死亡计数和剩余队列。
→
进入下一波倒计时 核心是 versus_special_respawn_interval 配置的刷特进程，例如 16 秒；anti-baiter 只在这个等待区间观察。
→
每秒检测推进状态 难度检测时间只决定何时开始评估 pressure；到基础刷特时间前，只观察，不开波。

正常推进路径

生还有推进 平均或最高 flow 持续增加，超过进度阈值。
→
保持正常倒计时 anti-baiter 不拦截，不额外改变刷特节奏。
→
配置时间到点开波 达到 16 秒这类基础刷特时间后，下一波按正常节奏开始。

anti-baiter 路径

生还不推进 处理完特感后卡点等刷，队形仍抱团互相保护。
→
进入 Pressure 检测时间到了且玩家抱团卡点，标记“到点后不要白给下一波”。
→
等机会再开波 必须先达到基础刷特时间；之后等生还移动、散开或达到兜底放行条件。
不是独立刷特 anti-baiter 挂在同一个下一波计时器上，只影响“到点后这一次下一波要不要按点放出”。
检测不是开波 6、8、10、12、16 秒这类难度时间是开始判断 pressure 的窗口，不是最早刷特时间。
下限必须守住 “等机会再开波”也不能小于 versus_special_respawn_interval；例如配置 16 秒时，任何 release 都不早于 16 秒。
